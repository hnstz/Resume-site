## Как запустить проект

### Как настроить реальную отправку писем (Опционально)
*Если переменные EMAIL_USER и EMAIL_PASS не заполнены в .env, сервер автоматически перейдет в режим мокирования (Dev Mode). Вместо реальной отправки писем, их содержимое будет безопасно выведено в логи терминала бэкенда для проверки логики работы.*

Для реальной отправки писем потребуется указать  данные почты в `.env`. Из соображений безопасности **никогда не используйте свой основной пароль от почты**. Вместо этого сгенерируйте специальный «Пароль приложения».

**Инструкция для Gmail:**
1. Перейдите в настройки вашего [Google Аккаунта](https://myaccount.google.com/).
2. Откройте раздел **«Безопасность»**.
3. Убедитесь, что у вас включена **Двухэтапная аутентификация** (без нее пароли приложений недоступны).
4. В строке поиска настроек аккаунта (сверху) введите **«Пароли приложений»** (App passwords) и перейдите в этот раздел.
5. Укажите любое понятное название (например, `Node Mailer` или `Portfolio`) и нажмите **«Создать»**.
6. Скопируйте появившийся 16-значный код (без пробелов) и вставьте его в файл `.env` в переменную `EMAIL_PASS`.
7. В переменную `EMAIL_USER` впишите адрес этой же почты.

После этого сервер сможет отправлять реальные письма, используя ваш аккаунт как почтовый сервер.

Перед сборкой создайте файл `.env` в корневой директории проекта со следующими переменными:
```text
EMAIL_USER=ваш_email
EMAIL_PASS=пароль_приложения_email
OPENAI_API_KEY=ваш_ключ_openai
```

### Запуск через Docker (Рекомендуемый способ)
Для запуска на вашем компьютере должны быть установлены Docker и Docker Compose. 


#### 1. Склонируйте репозиторий:

```bash
git clone [https://github.com/hnstz/Resume-site.git](https://github.com/hnstz/Resume-site.git)
cd Resume-site
```
#### 2. Выполните команду для сборки и запуска контейнеров:

```bash
docker-compose up -d --build
Откройте приложение в браузере:
```

Фронтенд (Nginx): http://localhost


Бэкенд API (Express): http://localhost:3000

### Локальный запуск (Без Docker)
Для запуска потребуются установленные Node.js и npm. Файл .env необходимо поместить в папку backend.

#### 1. Запуск бэкенда:

``` bash
cd backend
npm install
npm start
```

#### 2. Запуск фронтенда:

``` bash
cd frontend
npm install
npm run dev
```

## Стек
Frontend: React, TypeScript, Vite, SCSS (CSS-modules, BEM-методология).

Backend: Node.js, Express, Nodemailer, OpenAI API.

Инфраструктура: Docker, Docker Compose, Nginx.

Архитектура:

Проект построен по клиент-серверной архитектуре с разделением на SPA (Single Page Application) и REST API бэкенд.
Система полностью контейнеризирована:

1) Фронтенд компилируется в статические файлы и раздается через веб-сервер Nginx.
   
2) Бэкенд работает в изолированном Node.js контейнере, обрабатывая запросы формы и логику AI-ассистента.

## Как реализована форма
Форма обратной связи реализована с использованием контролируемых компонентов React.

На клиенте: Осуществляется сбор данных (имя, почта, текст), блокировка отправки пустых полей, обработка состояния загрузки и отправка HTTP POST-запроса на сервер.

На сервере: Node.js (через Express) принимает запрос, валидирует данные и использует библиотеку Nodemailer совместно с SMTP-сервером для безопасной пересылки содержимого формы на электронную почту владельца.
